JavaScript Charting capabilities

Starting FusionCharts/PowerCharts v3.2, you can now render the charts on devices like iPhone/iPad using the in-built JavaScript rendering capabilities, powered by Highcharts library. The charts rendered by this module are purely JavaScript based and do not need Flash Player at all. FusionCharts license agreement includes a license for HighCharts, and as such you do not need to license it separately.

FusionCharts gives you the option of either using automatic fallback, wherein the charts figure out the best mode of rendering (either Flash or JavaScript, based on availability of Flash Player), or lets you configure whether to render charts exclusively using JavaScript. If you are already using FusionCharts and wish to add iPad/iPhone fallback, you just need to replace the old SWF and JS files with the new ones - without changing a single line of code.

In case you wish to render JavaScript based charts only, irrespective of whether Flash Player is installed or not, all you need to do is add a single line of code to your existing implementation. This has been covered in forthcoming sections. Meanwhile, let's get an overview of FusionCharts JavaScript based charting capabilities.

How it works?

FusionCharts v3.2 features a completely new FusionCharts JavaScript class. FusionCharts JavaScript class now consists of three main files: FusionCharts.js, highcharts.js and jquery.min.js. These three files are present in Download Pack > Charts folder. Although you only need to include FusionCharts.js in your web page, highcharts.js and jquery.min.js must be present in the folder that contains FusionCharts.js. FusionCharts.js dynamically loads the other JavaScript on-demand (i.e., if and when needed).

When FusionCharts class determines that the device is iPad/iPhone, it automatically switches the rendering mode to JavaScript.

However, you can also forcefully set the rendering method to JavaScript from your code. For this you need to add a single line - FusionCharts.setCurrentRenderer('javascript') before you render a chart.

Additionally, you can setup the FusionCharts class to switch the rendering mode to JavaScript whenever Flash Player is not installed for the browser. For this you just need to call FusionCharts._fallbackJSChartWhenNoFlash() before you render a chart.

Limitations

As of now, the FusionCharts JavaScript renderer has the following limitations:

  • Trendlines values in all JavaScript charts do not get shown on the right-hand side of the chart
  • Dashed lines and borders are not supported in all JavaScript charts
  • Zero thickness value in Flash based charts produces a hair-line, while JavaScript based charts do not draw the line
  • plotFillAlpha='0' in Flash charts hides the plot border, it does not hide the plot boder in JavaScript charts. To hide the border, one needs to specify plotBorderAlpha='0' as well
  • Interactivity (Drag or Select) is not supported in Drag Node, Drag Column, Drag Line, Drag Area and Select Scatter charts
  • Gradient rendering in JavaScript charts is not supported
  • Color palettes (1-5) in JavaScript charts are not supported
  • Server-side export (as image/PDF) of JavaScript charts is limited only to download
  • Client-side export and client-side batch export are not supported by JavaScript charts
  • JavaScript based Candlestick chart does not support export features
  • Anchors in Radar chart which fall behind another are not clickable (while using link feature or showing tooltips)
  • Long labels in Drag Node chart can not be wrapped
  • Divline related attributes in Multi-axis Line chart do not work if specified in <axis> element
  • The hierarchical hover effect does not work in Multi-Level Pie chart
  • In Waterfall chart the connector-lines start from center of column
